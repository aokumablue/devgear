# チェックポイント保存・再開スキル

## 発動タイミング

- 10 ステップ以上の反復ループ開始前
- 「途中から再開したい」「チェックポイントを作って」と言われたとき
- レート制限中断が予想されるセッション（記事生成ループ / テストゲートリファクタリング）
- `session_end.py` がメッセージ数 30 件超で自動保存したチェックポイントを引き継ぐとき

## 保存先

`~/.devgear/session-data/checkpoint-<YYYY-MM-DD>-<slug>.md`

## チェックポイントファイルのフォーマット

```
---
task: <タスク名（20文字以内）>
completed: false
---

## 目標
<最終ゴールを1〜2文で>

## 完了済みステップ
- [x] ステップ1: ...
- [x] ステップ2: ...

## 進行中
- [ ] ステップN: ...（現在ここ）

## 残りステップ
- [ ] ステップN+1: ...
- [ ] ステップN+2: ...

## 変更済みファイル
- path/to/file1.py
- path/to/file2.md

## 再開コンテキスト
<次のセッションで必要な最小限のコンテキスト。500文字以内。>
```

## 操作

### 保存（新規）
1. 上記フォーマットでファイルを作成する
2. `slug` は タスク名をケバブケースに変換したもの（例: `checkpoint-2026-05-09-article-loop.md`）
3. `completed: false` で保存

### 更新（各反復後）
1. 完了したステップを `[x]` に更新する
2. 「進行中」を現在のステップに更新する
3. 「変更済みファイル」を追記する

### 完了マーク
タスク完了時は `completed: false` を `completed: true` に変更する。`true` のチェックポイントは次セッションで自動注入されない。

### 再開
「前のチェックポイントから再開して」と言われたら:
1. `~/.devgear/session-data/checkpoint-*.md` で `completed: false` のファイルを検索
2. 最新のものを読み込み、「進行中」ステップから作業を再開する
