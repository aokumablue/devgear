# 仕様徹底確認

## 発動タイミング

- command/skill 開始時に方針・制約・成功条件が未確定
- 「徹底的に質問して」「設計を問いただして」
- 計画・仕様・運用方針のストレステスト

## 進行ルール

1. 論点（目的・制約・成功条件・非目標）を先に列挙
2. 論点ごとに質問、回答が曖昧なら分岐を掘り下げ
3. コードベース調査で確定できる回答は調査してから次へ
4. 各質問に推奨回答案と判断理由を短く添える
5. 未解決項目がある限り実行確定に進まない

## 終了条件

- 全論点が「決定」または「保留（理由つき）」
- 実行可否と次アクションが一意に判断できる
- ユーザーの明示承認（「進めて」「OK」等）取得
