# TDD コマンド

**a-tdd** エージェントを起動してTDD強制。

## 永続メモリ

search: `test {機能ドメイン}` (file_pattern: *test*.py) / `bug fix error {関連キーワード}`
record: `{"event_type": "tdd", "content": "{テストサマリー: X tests, Y% coverage}"}`
参照: テストテンプレート / クリティカルコード検出 / カバレッジ傾向

## 使うタイミング

新機能実装・新fn/コンポーネント追加・バグ修正（まず再現テスト）・既存コードリファクタ・重要ビジネスロジック

## TDD サイクル

```
RED → GREEN → REFACTOR → REPEAT
```

1. IFを定義（SCAFFOLD）
2. 失敗するテスト書く（RED）
3. テスト実行・失敗確認
4. 最小限実装（GREEN）
5. テスト実行・合格確認
6. コード改善（REFACTOR）
7. テスト再実行・グリーン確認
8. カバレッジ確認

## DO / DON'T

- DO: 実装前にテスト書く / テスト実行して失敗確認 / 最小コード書く / 振る舞いをテスト
- DON'T: テストより先に実装 / 各変更後テスト未実行 / 一度に大量コード / 実装詳細をテスト

## カバレッジ要件

- 全コード: リポジトリの目標率（`CLAUDE.md`）
- 金融計算・認証・セキュリティ・コアビジネスロジック: **100%必須**

## 連携

- 実装前: `/c-plan` で何を作るか確認
- 実装後: `/c-review` でレビュー
